  • # Mysql 索引

    索引是MySQL的重要功能。它支持不同的类型索引，比如：主键索引，唯一索引，普通索引，全文索引。索引有助于加快从MySQL数据库服务器检索数据的速度。当从数据库表中检索数据的时候，MySQL首先检查表的索引是否存在；如果存在则将使用索引选择确切的物理对应行，而无需扫描整个表。

    # 索引的优缺点

    # 优点

    • 加快数据库服务器检索数据的速度。
    • 主键索引和唯一索引有助于避免重复的行数据。
    • 用户有机会针对这样索引的任何字段中的大量文本进行优化搜索。

    # 缺点

    • 占用磁盘空间，创建索引时会创建单独文件（该文件仅存储要排序的字段）。
    • 索引还影响操作（比如INSERT,UPDATE,DELETE）的速度（必须付出性能代价，因为每次记录更改时，索引都必须更新）。

# 索引类型

索引名称 关键词 特点
主键索引 PRIMARY KEY 一个表中只有一个主键索引，这一列的值是唯一的且不能为NULL
唯一索引(unique) UNIQUE 这一列的值是唯一不能重复的
普通索引(Normal) INDEX 提高查询速度
全文索引(fulltext) FULLTEXT 全文索引在大量的数据面前，能比like+%快N倍
